# RAGU-lm

RAGU-lm — компактная дообученная модель (Qwen-3-0.6B), специализированная для
извлечения структурированной информации из русскоязычных текстов. Модель
обучена на датасете NEREL и используется через адаптер `RaguLmArtifactExtractor`.

> RAGU-lm предназначен для **русскоязычных** корпусов. Для других языков
> используйте LLM-экстракторы (`ArtifactsExtractorLLM`, `TwoStageArtifactsExtractorLLM`).
> Обзор см. в [`ragu_components.md`](ragu_components.md), API — в
> [`../../ragu/triplet/README.md`](../../ragu/triplet/README.md).

Модель обучена решать четыре задачи:

1. Распознавание ненормализованных именованных сущностей из текста.
2. Нормализация каждой из ненормализованных сущностей относительно текста.
3. Генерация определения каждой из нормализованных именованных сущностей относительно текста.
4. Генерация определения направленного отношения между парой нормализованных именованных сущностей относительно текста.

---

## Промпты

**Системный промпт:**

```
Вы - эксперт в области анализа текстов и извлечения семантической информации из них.
```

**1. Извлечение сущностей:**

```
Распознайте все именованные сущности в тексте и выпишите их список с новой строки.\n\nТекст: {input_text}\n\nИменованные сущности:
```

**2. Нормализация:**

```
Выполните нормализацию именованной сущности, встретившейся в тексте.\n\nИсходная (ненормализованная) именованная сущность: {source_entity}\n\nТекст: {source_text}\n\nНормализованная именованная сущность:
```

**3. Генерация описания сущности:**

```
Напишите, что означает именованная сущность в тексте, то есть раскройте её смысл относительно текста.\n\nИменованная сущность: {normalized_entity}\n\nТекст: {source_text}\n\nСмысл именованной сущности:
```

**4. Извлечение отношений:**

```
Напишите, что означает отношение между двумя именованными сущностями в тексте, то есть раскройте смысл этого отношения относительно текста (либо напишите прочерк, если между двумя именованными сущностями отсутствует отношение).\n\nПервая именованная сущность: {first_normalized_entity}\n\nВторая именованная сущность: {second_normalized_entity}\n\nТекст: {source_text}\n\nСмысл отношения между двумя именованными сущностями:
```

**Вход:** `Исходная (ненормализованная) именованная сущность: пресс-секретарю` + исходный текст.

**Вход:** `Именованная сущность: пресс-секретарь` + исходный текст.


**Вход:**

**Ожидаемый ответ:**

```text

```

## Как использовать

Для инференса RAGU-lm рекомендуется использовать [vLLM](https://docs.vllm.ai/en/latest/).

Запуск сервера:

```bash
vllm serve RaguTeam/ragu-lm --max_model_len 4096
```

Инициализируем класс `RaguLmArtifactExtractor` и далее можем обращаться к модели в коде:

```python
from ragu.chunker.types import Chunk
from ragu.models.llm import LLMOpenAI
from ragu.models.openai import CachedAsyncOpenAI
from ragu.triplet.ragu_lm_artifact_extractor import RaguLmArtifactExtractor

client = CachedAsyncOpenAI(
    base_url="http://0.0.0.0:8000/v1/",
    api_key="dummy-api-token",
)
llm = LLMOpenAI(
    client=client,
    model_name="RaguTeam/ragu-lm",
)

pipeline = RaguLmArtifactExtractor(
    llm=llm,
)

chunks = [Chunk(content="Текст источника.", chunk_order_idx=0, doc_id="doc-1")]
entities, relations = await pipeline.extract(chunks)
```
